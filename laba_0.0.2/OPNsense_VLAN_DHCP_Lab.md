# Лабораторна робота: Налаштування VLAN та Kea DHCP в OPNsense

> **Мета:** Навчитися створювати VLAN-інтерфейси та налаштовувати Kea DHCPv4 сервер на брандмауері OPNsense.

---

## Частина 1: Створення VLAN пристроїв

### Крок 1.1 — Відкрити налаштування VLAN

1. Перейдіть у меню: **Interfaces → Other Types → VLAN**  
   *(або Interfaces → Devices → Settings, якщо "Other Types" відсутній)*
2. Натисніть кнопку **`+`** (Add) для додавання нового VLAN.

### Крок 1.2 — Заповнити поля VLAN (приклад для VLAN 10)

| Поле | Значення |
|------|----------|
| Device | `vlan010` *(підставляється автоматично)* |
| Parent | `vtnet0` *(ваш LAN-інтерфейс)* |
| VLAN tag | `10` |
| Description | `VLAN_STUDENT_10` |

> 📷 *Дивіться скріншот: Edit Vlan — поля Device, Parent, VLAN tag*

Натисніть **Save**.

### Крок 1.3 — Повторити для інших VLAN

Повторіть кроки 1.1–1.2 для:
- **VLAN 20** (tag: `20`, device: `vlan020`)
- **VLAN 30** (tag: `30`, device: `vlan030`)
- **VLAN 40** (tag: `40`, device: `vlan040`) *(за потреби)*

---

## Частина 2: Призначення інтерфейсів (Assignments)

### Крок 2.1 — Відкрити Assignments

1. Перейдіть у **Interfaces → Assignments**.
2. Знизу сторінки знайдіть розділ **"Assign a new interface"**.

### Крок 2.2 — Додати VLAN-інтерфейс

1. У полі **Device** оберіть зі списку ваш VLAN-пристрій, наприклад:  
   `vlan040 Vlan040 (Parent: vtnet0, Tag: 40)`
2. У полі **Description** введіть назву (необов'язково).
3. Натисніть **Add**.

> 📷 *Дивіться скріншот: "Assign a new interface" — вибір vlan040*

Після додавання в таблиці з'явиться новий рядок (наприклад, `OPT4`). Повторіть для всіх VLAN.

### Крок 2.3 — Результат призначень

Після додавання всіх VLAN таблиця має виглядати так:

| Interface | Identifier | Device |
|-----------|-----------|--------|
| [LAN] | lan | vtnet0 |
| [OPT4] | opt4 | vlan040 (Parent: vtnet0, Tag: 40) |
| [Vlan10] | opt1 | vlan010 (Parent: vtnet0, Tag: 10) |
| [Vlan20] | opt2 | vlan020 (Parent: vtnet0, Tag: 20) |
| [Vlan30] | opt3 | vlan030 (Parent: vtnet0, Tag: 30) |
| [WAN] | wan | vtnet1 |

> 📷 *Дивіться скріншот: таблиця Assignments із усіма інтерфейсами*

---

## Частина 3: Активація інтерфейсу та налаштування IP

### Крок 3.1 — Відкрити налаштування інтерфейсу

1. У таблиці Assignments натисніть на назву нового інтерфейсу (наприклад, `OPT4`).
2. Відкриється сторінка **Basic configuration**.

### Крок 3.2 — Базові налаштування

- Поставте галочку **Enable Interface**.
- Переконайтесь, що **Identifier** відповідає вашому VLAN (наприклад, `opt4`).
- **Device** має показувати правильний VLAN-пристрій (наприклад, `vlan040`).
- У полі **Description** введіть зручну назву: `Vlan040`.

> 📷 *Дивіться скріншот: Basic configuration — Enable, Identifier opt4, Device vlan040*

### Крок 3.3 — Налаштування статичного IP

Прокрутіть сторінку нижче до розділу **Static IPv4 configuration**:

| Поле | Значення |
|------|----------|
| IPv4 Configuration Type | `Static IPv4` |
| IPv4 address | `10.10.40.1` |
| Маска підмережі | `/24` |
| IPv4 gateway rules | `Disabled` |

> 📷 *Дивіться скріншот: Static IPv4 configuration — адреса 10.10.40.1/24*

Натисніть **Save**, а потім у верхній частині сторінки — **Apply changes**.

---

## Частина 4: Налаштування Kea DHCP

### Крок 4.1 — Увімкнення сервісу

1. Перейдіть у **Services → Kea DHCP → Kea DHCPv4**.
2. На вкладці **Settings** переконайтеся:
   - **Enabled** — галочка встановлена ✓
   - **Interfaces** — обраний потрібний інтерфейс (наприклад, `LAN`)
   - **Valid lifetime** — `4000` (секунд)
   - **Firewall rules** — галочка встановлена ✓
   - **Socket Type** — `raw`
3. Натисніть **Apply**.

> 📷 *Дивіться скріншот: Kea DHCPv4 Settings — Enabled, Interface LAN*

> ⚠️ **Увага:** Якщо сервіс не запускається, переконайтесь, що старий ISC DHCP вимкнено  
> *(Services → ISC DHCPv4 → знімаємо галочку Enable)*

### Крок 4.2 — Перегляд існуючих підмереж

Перейдіть на вкладку **Subnets**. Ви побачите список вже налаштованих підмереж:

| Subnet | Description | Pools |
|--------|-------------|-------|
| 10.10.10.0/24 | VLAN10 | 10.10.10.100 – 10.10.10.200 |
| 10.10.20.0/24 | VLAN20 | 10.10.20.100 – 10.10.20.200 |
| 10.10.30.0/24 | VLAN30 | 10.10.30.100 – 10.10.30.200 |
| 192.168.1.0/24 | LAN | 192.168.1.100 – 192.168.1.200 |

> 📷 *Дивіться скріншот: Kea DHCPv4 Subnets — список підмереж*

### Крок 4.3 — Додавання нової підмережі (Subnet)

1. Натисніть кнопку **`+`** (Add).
2. Заповніть поля:

| Поле | Значення (приклад для VLAN 10) |
|------|-------------------------------|
| Subnet | `10.10.10.0/24` |
| Description | `VLAN10` |
| Pools | `10.10.10.100 - 10.10.10.200` |
| Match client-id | ✓ (встановити галочку) |

### Крок 4.4 — Налаштування DHCP Option Data

У тому ж вікні розгорніть розділ **DHCP option data** і заповніть:

| Параметр | Значення |
|----------|----------|
| Routers (gateway) | `10.10.10.1` |
| DNS servers | `8.8.8.8` |

> 📷 *Дивіться скріншот: Edit Subnet — поля Subnet, Pools, Routers, DNS servers*

Натисніть **Save**, потім **Apply**.

---

## Частина 5: Dnsmasq (додатково)

Якщо в системі використовується **Dnsmasq DNS & DHCP** *(Services → Dnsmasq DNS & DHCP)*:

- Переконайтеся, що **Enable** вимкнено (щоб не конфліктувати з Kea DHCP), або налаштуйте лише DNS-частину.
- **Listen port** може бути змінений (наприклад, `53053`) для уникнення конфліктів.
- Підтримується **DHCP FQDN** та **DHCP default domain** (`internal`).

> 📷 *Дивіться скріншот: Dnsmasq DNS & DHCP — General settings*

## Підсумок

Після виконання всіх кроків ваша конфігурація має виглядати так:

```
OPNsense Firewall
├── LAN (vtnet0)           → 192.168.1.1/24,  DHCP: 192.168.1.100-200
├── VLAN10 (vlan010)       → 10.10.10.1/24,   DHCP: 10.10.10.100-200
├── VLAN20 (vlan020)       → 10.10.20.1/24,   DHCP: 10.10.20.100-200
├── VLAN30 (vlan030)       → 10.10.30.1/24,   DHCP: 10.10.30.100-200
└── VLAN40 (vlan040)       → 10.10.40.1/24    (за потреби)
```
