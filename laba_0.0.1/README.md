# 🔐 OPNsense: Доступ через WAN — Покрокова інструкція

> Інструкція описує, як правильно і безпечно налаштувати доступ до веб-інтерфейсу OPNsense через WAN-інтерфейс (наприклад, коли WAN має адресу з діапазону `192.168.88.x`).

У цій лабораторній роботі ви налаштуєте доступ до веб-інтерфейсу OPNsense через WAN-інтерфейс. Проблема полягає в тому, що локальна мережа ізольована — до неї має доступ лише віртуальна машина з консольним терміналом, тому керувати OPNsense через веб-інтерфейс з локальної мережі неможливо. Саме тому ми підключатимемося через WAN-інтерфейс, а щоб це стало можливим — спочатку скористаємося Shell для тимчасового вимкнення фаєрволу і виконаємо необхідні налаштування.

---

## Як отримати доступ до Shell

1. Перейдіть у Proxmox, виберіть вузол pve та у списку віртуальних машин натисніть на opnsense-firewall
![alt text](<image/image_01.png>)
![alt text](<image/image_12.png>)
2. Перейдіть у вкладку Console та запустіть віртуальну машину кнопкою Start (або "Включити VM")
3. Після завантаження авторизуйтесь: у полі Login введіть root, у полі Password введіть student
4. Після входу ви побачите меню опцій — введіть 8 та натисніть Enter, щоб отримати доступ до Shell

![alt text](<image/image_06.png>)

## Крок 1. Вимкніть фаєрвол

Якщо ви раніше не вимикали фаєрвол, скористайтесь командою `pfctl -d` — його необхідно перевести в неактивний стан, інакше система не дозволить доступ через WAN-інтерфейс.

**Введіть команду через Shell (консоль):**
```bash
pfctl -d
```
![alt text](<image/image_08.png>)
**Зверніть увагу:**

При натисканні кнопки **Apply Changes** у будь-якому розділі — це автоматично активує пакетний фільтр, тобто система блокує доступ через WAN-інтерфейс. Щоб мати повторний доступ до веб-інтерфейсу, треба кожного разу після **Apply Changes** виконувати команду **через Shell** `pfctl -d`.

---

## Крок 2. Дозвіл для приватних мереж ⚠️ (Найважливіше)

Оскільки WAN-інтерфейс має адресу з приватного діапазону (`192.168.88.x`), OPNsense за замовчуванням блокує такий трафік як підозрілий.

1. Перейдіть у **Interfaces → [WAN]**
2. Прокрутіть сторінку до розділу **Generic configuration**
3. **Зніміть галочки** з обох пунктів:
   - `Block private networks`
   - `Block bogon networks`
![alt text](<image/image_09.png>)
4. Натисніть **Save**, потім **Apply Changes**, потім ще раз у консолі:
```bash
pfctl -d
```

---

## Крок 3. Налаштування постійного правила WAN

Створіть правило фаєрволу, що дозволяє доступ до веб-інтерфейсу.

1. Перейдіть у **Firewall → Rules → WAN**
2. Створіть нове правило або відредагуйте наявне з такими параметрами:

| Параметр | Значення |
|---|---|
| Action | `Pass` |
| Direction | `in` |
| TCP/IP Version | `IPv4` |
| Protocol | `TCP` |
| Source | `any` *(або конкретний IP вашого ПК)* |
| Destination | `WAN address` |
| Destination Port Range | `HTTPS (443)` |
![alt text](<image/image_03.png>)


![alt text](<image/image_02.png>)
3. Збережіть правило та **перетягніть його на самий верх** списку (вище за всі інші правила)
повторіть те саме і для порту `(8443)` це ми робимо для подалших дій щоб збільшити безпеку та унеможливти проблеми з доступом. 
![alt text](<image/image_04.png>)

Чому ми створюємо ще нове правило для порту 8443? У кроці 5 треба буде замінити 443 на 8443 для підвищення безпеки і є імовірність що ми не зможемо отримати доступ до веб-інтефейсу Administration і щоб цього не було ми завчасно додаємо правило яке нам дозволить доступ і по 8443 порту

![alt text](<image/image_05.png>)
4. Натисніть **Apply Changes**, потім ще раз у консолі:
```bash
pfctl -d
```

---

## Крок 4. Виправлення проблеми "Reply-to"

Ця проблема проявляється так: сторінка авторизації відкривається, але після введення логіна/пароля з'єднання обривається або виникає помилка.

1. Перейдіть у **Firewall → Settings → Advanced**
2. Знайдіть опцію **Disable reply-to**
3. **Поставте галочку** — це дозволить фаєрволу коректно відповідати клієнтам, що знаходяться в тій самій підмережі
4. Натисніть **Save**. Якщо доступ заблокувало, виконайте у консолі:
![alt text](<image/image_10.png>)
```bash
pfctl -d
# а потім
pfctl -e
```

---

## Крок 5. Підвищення безпеки (Рекомендація) 🛡️

> Тримати адмін-панель на стандартному порту `443` на WAN-інтерфейсі — погана практика з точки зору безпеки.

**Зміна порту адміністрування:**

1. Перейдіть у **System → Settings → Administration**
2. Знайдіть поле **TCP port**
3. Змініть значення з `443` на інший порт, наприклад `8443`
4. Натисніть **Save**
5. Поверніться до **Кроку 3** та оновіть правило фаєрволу — замість порту `443` вкажіть новий порт `8443`
![alt text](<image/image_11.png>)
Після цього адмін-панель буде доступна за адресою:
```
https://192.168.88.X:8443
```

---

## ✅ Перевірка після налаштування

Після виконання всіх кроків рекомендується перевірити доступ:

- Відкрийте браузер у **режимі інкогніто** або спробуйте **інший браузер**
- Перейдіть за адресою `https://192.168.88.X:8443` (або `:443`, якщо не змінювали порт)
- Доступ має зберігатися навіть **після перезавантаження** віртуальної машини

---

## 📋 Підсумок кроків

| # | Що робимо | Де |
|---|---|---|
| 1 | Вимикаємо фаєрвол | Shell (`pfctl -d`) |
| 2 | Знімаємо блокування приватних мереж | Interfaces → WAN |
| 3 | Створюємо дозвільне правило | Firewall → Rules → WAN |
| 4 | Вмикаємо Disable reply-to | Firewall → Settings → Advanced |
| 5 | Змінюємо порт адміністрування | System → Settings → Administration |

---

> **Примітка:** Якщо ви хочете обмежити доступ до адмін-панелі лише з конкретного IP-адреса вашого ПК — на кроці 3 у полі **Source** оберіть `Single host` та вкажіть ваш IP. Це значно підвищить рівень безпеки.
